Linux security

Posted on 2016-04-23   |   In Linux   |  

Today I have learned how to manage softwares and users on a Linux system to increase system security.

It is noticeable that when we log in our VM we are logged in as a standard user. As a standard user, you cannot execute some of the Linux commands. If you want to run some commands that are limited to system administrators, you can prepend the word ‘sudo’ at the beginning of your commands.

Command sudo ls -al /home/ubuntu/.ssh would list all files in the .ssh directory.

So now we come to the next stage of installing softwares on Linux. The following commands are useful in terms of installing and updating softwares on the system.

  • Use command cat /etc/apt/sources.list to get a list of software repositories.

  • Use command sudo apt-get update to update the package source list.

  • Use command sudo apt-get upgrade to update the softwares in the operation system.

  • Use command sudo apt-get autoremove to remove anything that is useless in the system.

  • Use command sudo apt-get install softwareName to install a software to the system.

Another important file in the system is ‘/etc/passwd’, which keeps track of all the users in the system.

  • Use command cat /etc/passwd to peek inside the file.

Each row in the passwd file represents a user in the following format:

  • username:password:UID:GID:UID info:home directory:command/shell

User management plays an important role in terms of system security. Here are some commands related to user management.

  • Use command sudo adduser userName to add a new user.

Once you have added a new user, you can use the following command to log in the system as the new user.

  • ssh userName@127.0.0.1 -p 2222

Let’s break this command down a bit.

ssh: ssh is the application we use to connect to a server
127.0.0.1: this is the IP we want to connect to
-p 2222: this is the port that we use to connect to the IP

Now we need to give the new user sudo access. Let’s assume we would like to create a new user named dennis.

  1. copy a sudoer file and name it as the new user
    sudo cp /etc/sudoers.d/vagrant /etc/sudoers.d/dennis

  2. edit the new user sudoer file using your preferred editor
    sudo nano /etc/sudoers.d/dennis

  3. change the user in the sudoer file
    dennis All=(All) NOPASSWD:ALL

After adding a new user, you can force the new user password to expire by using the following command:

  • sudo passwd -e userName

There is another authentication method, key-based authenticaton, that is more secure than using password.

Basic Markdown

Since I am using Hexo framework for my blog, I need to have a quite review of some basic markdown syntax.

  • To create header, put 1-6 “#” at the beginning of the header text to represent h1 to h6
  • To create blockquote, put ‘<’ at the beginning of the text
  • To create italic emphasis, use ‘*’
  • To create bold emphasis, use ‘**’
  • To create an unordered list, use ‘-‘
  • To create an ordered list, use number followed by a period
  • To create a link, use [example link](http://example.com/)
  • To create an image, use ![alt text](/path/to/img.jpg “Title”)

So here are some basic markdown syntax I need to know for my daily blogging activities.

Intro to Linux

Posted on 2016-04-23   |   In Linux   |  

When people say Linux is free, have you ever considered if it’s free as in beer or free as in speech? The success of Linux comes from its free as in speech approach. You can virtually run, see, modify and distribute the operation system however you would like.

Some sort of Linux knowledge is necessary if you want to get into web development job. So let’s dive right into the Linux world!

Before we go deeper, it’s handy to know some Vagrant commands.

Previously we have learned to use ‘vagrant up’ to fire up the virtual machine and ‘vagrant ssh’ to get into secured shell. Here are some other commands that are useful.

  • ‘vagrant status’ is used to check on the status of the VM.
  • ‘vagrant suspend’ is used to suspend your VM, which means to put it into a sleep mode.
  • ‘vagrant halt’ is used to turn off your VM. (let’s call it a day and power off your viftual machine!)
  • ‘vagrant destroy’ is used to format your VM, and to roll it back to baseline Linux installation.

Now we are in the shell. It’s good to have some basic understanding of the important folders in your VM.

If you go to your home directory and type ‘ls -al’, a detailed list of directories should show up. Some of the folders here are really important and we need to get acquaintance with them.

  • ‘etc’ directory is where the configuration files live
  • ‘var’ directory is where you can find system and application logs
  • ‘bin’ directory is where executable binaries are stored
  • ‘sbin’ directory is where applications for system administrator sit, normal users typically don’t use

Here you may wonder why we can simply type ‘ls’ if all the commands live in ‘bin’ directory instead. This is where $PATH comes into play.

Try ‘echo $PATH’ to see your current PATH.

If you haven’t fiddled with your PATH setting much then you would expect to see something like this

/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games

When you type a command, the Linux would travel trough these directories one by one to see if the command is in there. You can edit this path at /etc/environment if it need be.

Basic Linux commands

Posted on 2016-04-20   |   In Linux   |  

I have just set up a virtual machine (VM) in my own computer by using Virtualbox and Vagrant today.

Virtualbox is lightweight compared to VMware but it gives me enough power to experience things around a Linux system. Vagrant is a software that simplifies the configuration of my VM environment, so I can easily set up my virtual environment in a few commands.

After installing the Virtualbox and Vagrant, and placing the Vagrant configuration file into my workspace folder. It is ready to boot up my VM! Hooray!

Well, just a few commands away before I can see my lovely VM up and running!

First navigate to the folder where the Vagrant configuration file sits, then execute the following two commands.

vagrant up: to start my VM
vagrant ssh: to get into the security shell

Now you are officially an otter because you will soon know how to rock a shell! (Geek humor)

A few Linux commands that I have just learned.

  • ls: list all the files in the current directory, use -a to show hidden files and directories, use -l to display more details
  • clear: clear all the output from the screen
  • curl http://xxxx.zip -o things.zip: download a file from the internet
  • date: return the current date
  • expr 2 + 2: it evaluates 2 + 2
  • echo: it is equal to the print method of the shell
  • echo ‘This is a test’ > test.txt: echo can also be used to create a file
  • uname / uname -a: prints the operating system’s name
  • hostname: prints the computer’s own name
  • host xxx.com: finds out IP address and other info
  • bash –version: tells you the version of your bash shell
  • history: tells you all the commands you run previously
  • exit: exit the VM
  • uptime: checks the run time of the system
  • unzip xxx.zip: unzip the file
  • cat file: short for concatenate, it also displays the content of the concatenated file
  • wc: word count, tells you how many lines, words and bytes are in a file
  • diff file1 file2: tells you the difference between two files
  • man command: see the manual of the command
  • rm: remove a file or a directory. Two very useful options are -f force and -r recursive
  • ping url/ip: test whether a machine on the internet is alive
  • sort: sort lines of txt file
  • bc: basic calculator
  • less: display text one page at a time
  • nano filename: edit a file
  • pwd: short for ‘print working directory’
  • cd: use to change directory
  • mv file directory: move a file to a directory or change a file name
  • cp file directory: copy a file
  • mkdir directory: create a new directory
  • rmdir directory: create an empty directory

Some key stroke combos that are awefully handy.

  • Ctrl + r: search for the command that you have used before
  • Ctrl + d: exit the VM
  • Ctrl + c: terminate a program

A few notes about Linux file system

  • Linux file system has only one root, so any file can be referred to using a full path.
  • Linux use ‘/‘ in its file path, for example, /home/docs/text.txt.
  • . means the current directory, .. means the parent directory, ~ means the home directory.
  • cd without argument would take you to home directory.
  • globbing patterns, also known as wildcards, are used to work with multiple files.

For binary search to work, the array must be sorted first.

Pseudo-code for binary search

get the min value
get the max value

while max value greater than and equal to min value

calculate the mid value

if the search value is greater than the mid value
    set min to mid + 1
if the search value is less than the mid value
    set max to mid - 1
else
    return mid value

return -1
