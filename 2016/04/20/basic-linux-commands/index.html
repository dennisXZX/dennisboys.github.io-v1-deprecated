Basic Linux commands

Posted on 2016-04-20   |   In Linux   |  

I have just set up a virtual machine (VM) in my own computer by using Virtualbox and Vagrant today. Yeah, it’s time to put some Linux skills into my pocket!

Virtualbox is kinda lightweight compared to VMware but it gives me enough power to experience things around in a Linux system. Vagrant is a software that simplifies the configuration of my VM environment, so I can easily set up my virtual environment in a few commands.

After installing the Virtualbox and Vagrant, and placing the Vagrant configuration file into my workspace folder. It is ready to boot up my VM! Hooray!

Well, just a few commands away before I can see my lovely VM up and running!

First navigate to the folder where the Vagrant configuration file sits, then execute the following two commands.

vagrant up: to start my VM
vagrant ssh: to get into the security shell

Note: Vagrant has helped us to shorten things a bit here. Actually under the hood, this command ssh userName@127.0.0.1 -p 2222) is executed.

Now I am officially an otter because I will soon know how to rock a shell! (Geek humor)

A few Linux commands that I have just learned.

  • ls: list all the files in the current directory, use -a to show hidden files and directories, use -l to display more details
  • clear: clear all the output from the screen
  • curl http://xxxx.zip -o things.zip: download a file from the internet
  • date: return the current date
  • expr 2 + 2: it evaluates 2 + 2
  • echo: it is equal to the print method of the shell
  • echo ‘This is a test’ > test.txt: echo can also be used to create a file
  • uname / uname -a: prints the operating system’s name
  • hostname: prints the computer’s own name
  • host xxx.com: finds out IP address and other info
  • bash –version: tells you the version of your bash shell
  • history: tells you all the commands you run previously
  • exit: exit the VM
  • uptime: checks the run time of the system
  • unzip xxx.zip: unzip the file
  • cat file: short for concatenate, it also displays the content of the concatenated file
  • wc: word count, tells you how many lines, words and bytes are in a file
  • diff file1 file2: tells you the difference between two files
  • man command: see the manual of the command
  • rm: remove a file or a directory. Two very useful options are -f force and -r recursive
  • ping url/ip: test whether a machine on the internet is alive
  • sort: sort lines of txt file
  • bc: basic calculator
  • less: display text one page at a time
  • nano filename: edit a file
  • pwd: short for ‘print working directory’
  • cd: use to change directory
  • mv file directory: move a file to a directory or change a file name
  • cp file directory: copy a file
  • mkdir directory: create a new directory
  • rmdir directory: create an empty directory
  • touch fileName: create an empty file

Some key stroke combos that are awefully handy.

  • Ctrl + r: search for the command that you have used before
  • Ctrl + d: exit the VM
  • Ctrl + c: terminate a program

A few notes about Linux file system

  • Linux file system has only one root, so any file can be referred to using a full path.
  • Linux use ‘/‘ in its file path, for example, /home/docs/text.txt.
  • . means the current directory, .. means the parent directory, ~ means the home directory.
  • cd without argument would take you to home directory.
  • globbing patterns, also known as wildcards, are used to work with multiple files.
